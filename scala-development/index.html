Scala development

Scala development at SoftwareMill

Scala is gaining popularity at SoftwareMill. We started a couple of years ago with a single project based purely on Scala, and now almost all new projects use Scala in some way, with over half of SoftwareMill coders knowing Scala very well.

 

Over and above being an expressive, scalable and fun language to write in, Scala has a rich ecosystem that is one of the main things which make it so great. The Typesafe stack and frameworks like Akka, Spray, Scalding or Scalatra make it much simpler and more efficient to achieve our daily development tasks.

 

Traditionally at SoftwareMill we try to give back to the community. We have been sharing our Scala experience at conferences such as JavaZone, Devoxx, Confitura and BuildStuff. We also run some open source projects (see below).

Scala projects

Scala is perfect for big projects, as it’s really productive. It enables developers to reduce the size of the source code by a factor of at least 2 or 3 compared to Java.

 

Case study: Yap.TV 

We developed the backend system for Yap.TV, a social TV app. The backend spans multiple servers, and is implemented using Akka and Spray. Used since the inception of the project, Scala has proved a powerful and concise language allowing us to create readable code and deliver features quickly at the same time. 

 

Case study: transforming data with Hadoop

We developed the backend for an analysis application that aggregates information regarding various customer behaviour patterns. Using this data, merchants are able to plan their marketing campaigns. To achieve the goals of this project, we used the Scala language together with Apache Hadoop and Apache Pig to transform the customer data, and Spray to serve the results.

Open source projects

A number of our open source projects are based on Scala. Firstly, there is Bootzooka –
a template web application, using modern technologies and a modern stack:


• Scalatra for creating REST APIs
• AngularJS, stylus, Twitter Bootstrap for the frontend
• joint SBT/Grunt build process
• Scala as the main backend development language
• Akka for asynchronous and background processing
• MongoDB as a persistent store

Bootzooka allows us to start a new project quickly, and deliver value to our customers as fast as possible. We are also running a series of smaller open source projects: Veripacks, ElasticMQ and MacWire. All written and leveraging the huge possibilities of Scala.
